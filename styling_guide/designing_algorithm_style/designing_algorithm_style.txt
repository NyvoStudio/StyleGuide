Black onion project algorithm writing style guide :

    1. Loop starting , Finishing , Change In operation's will be written In sqaure.

    2. Normal logical activites will be written In circle ( Not classes , Function's , Decorators )

    3. User Input's are shown in Hexagon shape.

    4. System Decisions And exception's will be written In rhombuse.

    5. Functions Will be written in triangles.

    6. Function Calls be written In Call Activity Shape ( Yellow Square ).

    7. Connected attribute's will be written In Page refrences.

    8. Database and class connection's will be written In Database Shape.

    9. Connection's will be Refrenced With circlehead Lines.

    10. Import's and accesse's will be Refrenced With Arrowhead Lines.

    11. Partition's will be shown In vertical Lines.

    12. Partition Name's will be shown in Horizontal Lane.

    13. Comment's will be sown In Comment_Alt shape.

    14. Ordering's Will be Written In brackets ( Number ) .

    15. All algorithms Must be written In visual paradigm online.


 قوانین فلوچارت نویسی پروژه آنین

    1. شروع شدن حلقه ها و پایان آنها ( شامل شروع و پایان برنامه هم خواهد شد ) به شکل مربع نشان داده خواهند شد ( Square ).
    
    2. عملکردهای منطقی و خارج از تابع ( فانکشن ) و کلاس ها به شکل دایره نشان داده خواهند شد ( Circle ).

    3. ورودی ها به شکل شش ضلعی نشان داده خواهند شد ( Hexagon ).

    4. انتخاب های سیستمی ، اکسپشن هندلینگ ها ( ایف ، الس ، ترای ، اگزپت ، فاینالی ، سوییچ ) به شکل لوزی نشان داده خواهند شد ( Rhombuse ).

    5. تابع ( فانکشن ها ) به شکل مثلث نشان داده خواهند شد ( Triangle ).

    6. صدا زدن تابع ها ، ( کال کردن فانکشن ها ) به شکل مربع زرد نشان داده خواهند شد ( Activity Shape ).

    7. ویژگی های عرضی به شکل پنج ضلعی صاف کف نشانه داده خواهند شد ( چسبیده به شکل اصلی ) ( In Page refrence ).

    8. دیتابیس ها ، فایل های بزرگ اطلاعاتی ، کلاس ها ، دیتاکلاس ها به شکل دایره سه بعدی نشان داده خواهند شد ( Database shape ).

    9. ارتباطات دخیل از جمیل وجود فانکشن و یا عملکرد در دیتابیس ، به شکل خط ارتباطی با سر دایره نشنان داده خواهند شد ( Circlehead lines ).

    10. واردات ( ایمپورت کردن ) ، رفتن برنامه به بخشی دیگر ، به شکل خط ارتباطی با سر پیکان نشان داده خواهند شد ( Arrowhead lines ).

    11. بخش بندی کد ( مانند جدا کردن فایل ها از یکدیگر ) به شکل خط عمودی نشان داده خواهند شد ( Vertical Line ).

    12. نام های بخش بندی ( مانند نام فایلی که بخش بندی به آن اشاره دارد ) به شکل لاین بندی افقی نشان داده خواهند شد ( Horizontal Lane ).

    13. کامنت ها و توضیحات اضافی به شکل چت نشان داده خواهند شد ( Comment_Alt Shape ).

    14. اولویت بندی ها به شکل گیومه قلاب نشان داده خواهند شد ( [] ).

    15. تک تک الگوریتم های تیم توسط برنامه ویژوال پارادایم آنلاین نوشته میشوند.
